# Requirements-Analysis
**Описание задания:**  
Проведен анализ требований для модуля "Регистрация и Авторизация" веб-приложения "Интернет-магазин" на основе пользовательских историй и макета (Figma).

**Цель:**  
Выявить несоответствия и дефекты в требованиях до начала разработки.

**Что было сделано:**  
- Проанализированы пользовательские истории:
- ID1 Регистрация пользователя / User Registration
- ID2 Вход в систему / User Login
- ID3 Выход из системы / User Logout
- ID4 Проверка авторизации перед доступом к функциональности / Authorization Check before Accessing Functionality
